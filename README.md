## 📌 TensorFlow vs PyTorch: 관계 정리


✅ 1. 둘 다 딥러닝 프레임워크 (공통점)

둘 다 Google(텐서플로우)과 Meta/Facebook(파이토치)이 만든 오픈소스 딥러닝 라이브러리.

이미지 분류, 자연어 처리, 음성 인식 등 모든 딥러닝 모델을 구현·학습 가능.

GPU, TPU를 활용해 빠르게 학습 가능.

즉, 같은 목적을 가진 라이벌.

🎯 2. 학습 방식 및 철학 차이 (가장 큰 차이)
⭐ PyTorch: “직관적·파이썬스러운” 방식

연구자들이 가장 많이 쓰는 이유.

코드가 직관적이고 디버깅이 쉬움.

연산이 바로 실행되는 동적 그래프(Eager Execution) 기반.

빠르게 실험하고 수정하기 좋음.

예) 실험, 논문 구현, 연구용에 최강.

⭐ TensorFlow: “산업·배포 최적화”

Google이 만들었기 때문에 대규모 시스템 배포에 강함.

TensorFlow Serving, TensorFlow Lite, TensorFlow.js 등 배포 생태계가 풍부.

고성능, 모바일·웹 배포에 강함.

예) 실제 서비스/프로덕션 환경에 자주 사용됨.

🔍 3. 관계를 비유로 표현하면

파이토치(Pytorch)는 연구실에서 사용하는 운동화
→ 가볍고 유연해서 실험하기 편함.

텐서플로우(TensorFlow)는 산업용 안전장비
→ 실제 공장/서비스에 투입하기 좋음, 대규모 배포 환경에 강함.

둘은 같은 목적을 위한 도구이지만 성격이 다르고, 많은 개발자들이 작업 목적에 따라 둘 중 하나를 선택하거나 둘 다 사용하기도 한다.

📌 4. 현재 흐름

최근 몇 년간 연구계/학계는 거의 80~90%가 PyTorch 중심
하지만 산업 분야에서는 TensorFlow + PyTorch가 반반 정도 사용됨.

✔️ 5. 요약
항목	TensorFlow	PyTorch
제작	Google	Meta(Facebook)
스타일	안정적·배포 중심	직관적·연구 중심
그래프	정적 그래프 중심	동적 그래프 중심
